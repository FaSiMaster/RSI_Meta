Fachbegriffe

9-Schritt-Methodik RSI
Normatives Verfahren pro Sicherheitsdefizit: Wichtigkeit (1) → Einzeichnen (2) → Abweichung (3) → Einzeichnen (4) → Relevanz SD (5) → Einzeichnen (6) → NACA (7) → Einzeichnen (8) → Unfallrisiko (9). Schritte 1, 3, 7 sind Benutzereingaben, die anderen Automatiken.
Wichtigkeit
Kategorie: gross / mittel / klein. Wie relevant ein Kriterium für Verkehrsablauf und Sicherheit ist. Kontextabhängig (io/ao). Quelle: WICHTIGKEIT_TABLE, 58 Kriterien.
Abweichung
Kategorie: gross / mittel / klein. Wie stark der Ist-Zustand von der Norm (VSS/SN) abweicht.
Relevanz SD
Kombiniertes Ergebnis Wichtigkeit × Abweichung. Wertebereich: gering / mittel / hoch.
Unfallschwere
Übersetzung NACA-Skala: 0–1 → leicht, 2–3 → mittel, 4–7 → schwer.
Unfallrisiko
Endbewertung Relevanz SD × Unfallschwere. Wertebereich: gering / mittel / hoch. Farbe: grün / orange / rot.
Verortung
Geometrische Defizit-Definition im 360°-Panorama. Typen: Punkt (mit Toleranz), Polygon (Eckpunkte), Gruppe (kombiniert).
Perspektive
Zusätzlicher 360°-Standort innerhalb einer Szene. Eigenes Panoramabild, eigener Startblick, eigene Verortungen (kein Fallback auf Haupt).
Standort-Navigation
Bidirektionale Verknüpfung zwischen Haupt-Panorama und Perspektiven via klickbare 3D-Marker.
Startblick
Anfangs-Blickrichtung beim Laden einer Szene/Perspektive (θ, φ).
Best-of-Punktesystem
Mehrfach-Durchgänge zulässig — bester Score zählt. Sterne: < 60 % (1), 60–89 % (2), 90–100 % (3).
Penalty / Strafabzüge
Kategorie falsch: −10 % auf Defizit. Hinweise (Lupe) aktiviert: −25 Pkt. pro Szene.
Kriterium
Einer von 58 Einträgen in WICHTIGKEIT_TABLE, bindet Kontext (io/ao) an Wichtigkeits-Wert.
Szene
Verkehrssituation mit Haupt-Panorama, optional Perspektiven, Defizit-Liste, Einstiegs-Text.
Thema / Unterthema
Thematische Szenen-Gruppierung (Oberthema → Unterthema → Szenen).
Kurs
Zeitlich begrenzter Zugang für Teilnehmergruppe: Zugangscode, optional Passwort, Gültigkeit, Themen-Auswahl.
Pseudonymisierung
Ersetzen eines identifizierenden Merkmals (Name) durch einen nicht direkt rückrechenbaren Wert (SHA-256-Hash). Nicht gleichbedeutend mit Anonymisierung (Art. 4(5) DSGVO).
